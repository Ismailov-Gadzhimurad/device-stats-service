# Device Stats Service

Сервис сбора и анализа статистики с устройств. Разработан в качестве решения тестового задания. 

## Реализованные требования

### Обязательные
- REST API на FastAPI
- Сохранение показаний устройства по `device_id` (формат `{"x": float, "y": float, "z": float}`)
- Анализ статистики за определенный период и за все время
- Числовые характеристики: минимум, максимум, количество, сумма, медиана
- Хранение данных в MySQL
- Контейнеризация: Docker + Docker Compose

### Опциональные
- Пользователи устройств и аналитика по пользователям
- Асинхронная аналитика (Celery)
- Нагрузочное тестирование (Locust) (выполнен)

Причина невыполнения опциональных требований - фокусировка на качестве обязательной части ТЗ.

# API
## Сохранить показания
**Запрос:**
```bash
curl -X POST "http://localhost:8000/api/v1/devices/1/stats" \
  -H "Content-Type: application/json" \
  -d '{"x": 10.5, "y": 20.3, "z": 30.7}'
```

**Ответ:**
```json
{
  "status": "ok",
  "device_id": "1"
}
```

## Получить аналитику
**Запрос:**
```bash
curl "http://localhost:8000/api/v1/devices/1/stats"
```

**Ответ:**
```json
{
  "device_id": "1",
  "min": 10.5,
  "max": 30.7,
  "count": 3,
  "sum": 61.5,
  "median": 20.3
}
```
